# M1 — Angular: Cart Service + Component

**Level:** Mid
**Skills tested:** Angular services, BehaviorSubject, component communication, subscription lifecycle
**Prerequisite:** J3 (or a working understanding of the cart concept)
**Files to edit:** `src/restaurant/menu/menu.component.ts`
**Files to create:**
- `src/restaurant/services/cart.service.ts`
- `src/restaurant/cart/cart.component.ts`
- `src/restaurant/cart/cart.component.html`
- `src/restaurant/cart/cart.component.css`

---

## Context

The J3 cart lives entirely inside `MenuComponent`. This means the cart resets whenever the user navigates away, and no other component can see the cart state. Extract the cart into a shared service and build a dedicated `CartComponent` to display it.

---

## Current state (after J3, or what you start with)

`MenuComponent` holds:

### Task 2 — Update MenuComponent
- Remove all local cart state (`cartItems`, `totalPrice`, `clearCart`, `addToCart` logic)
- Inject `CartService`
- Call `cartService.addItem(item)` in `addToCart()`
- Subscribe to `cart$` to show per-item badge counts (or use `async` pipe in template)
- Keep the "Add to Cart" button on each card

### Task 3 — CartComponent

### Task 4 — Integrate into AppComponent
- Import `CartComponent` in `AppComponent`
- Show a cart icon button in the toolbar (use `MatIconModule`, icon: `"shopping_cart"`)
- Add a `MatBadge` on the icon showing total item count (sum of quantities)
- Clicking the icon toggles a `MatDrawer` or `MatDialog` containing `<app-cart>`

---

## Acceptance criteria

- [ ] `CartService` is `providedIn: 'root'` with a single `BehaviorSubject` as source of truth
- [ ] `totalPrice$` is derived via `map`, not stored as a second subject
- [ ] Adding items in `MenuComponent` immediately reflects in `CartComponent`
- [ ] Quantity controls (+/−) work correctly; going below 1 removes the item
- [ ] Cart badge in toolbar shows the correct total item count
- [ ] No cart state lives in `MenuComponent` — it only calls service methods
- [ ] No memory leaks: subscriptions in components use `async` pipe or are unsubscribed
