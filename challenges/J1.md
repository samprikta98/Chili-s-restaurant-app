# J1 — CSS: Responsive Menu Grid

**Level:** Junior
**Skills tested:** CSS Grid, Flexbox, responsive breakpoints, transitions
**Files to edit:** `src/restaurant/menu/menu.component.css` and `src/restaurant/menu/menu.component.html`

---

## Context

The menu currently renders items stacked vertically — one card per full row. The layout is plain and doesn't adapt to screen size. Your task is to make it look polished and responsive without touching any TypeScript.

---

## Current code

**`menu.component.html`** (no changes needed here — unless you choose to adjust markup for layout):

---

## Tasks

### Task 1 — Responsive grid wrapper
Wrap the cards in a container (you may add a `<div class="menu-grid">` in the HTML) and style it so:
- **Desktop (>768px):** 3 columns
- **Tablet (≤768px):** 2 columns
- **Mobile (≤480px):** 1 column

Use CSS Grid with `auto-fill` or `auto-fit` + `minmax()` — a single rule should handle all breakpoints without media queries if done correctly.

### Task 2 — Card hover effect
Add a hover effect on each `.menu-item` card:
- Elevate the card slightly (`transform: translateY`)
- Increase the box shadow
- Animate smoothly with a CSS `transition` (no JavaScript)

### Task 3 — Price badge alignment
The price shown in `mat-card-subtitle` should always be right-aligned within the card header, regardless of how long the dish name is.

---

## Acceptance criteria

- [ ] Cards display in a 3 / 2 / 1 column grid at the respective breakpoints
- [ ] Hover lifts the card visually with a smooth transition (≤300ms)
- [ ] Price is right-aligned in the header on all screen sizes
- [ ] No JavaScript or TypeScript changes made
