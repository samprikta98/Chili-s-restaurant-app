## Context

The app uses Angular's default change detection strategy — every component re-checks on every event in the entire application. For small apps this is fine, but as the menu grows (imagine 50+ items) and more components are added, this becomes a bottleneck.

---

### Task 2 — Use `track` in `@for`
Migrate the menu template from `*ngFor` to the modern `@for` control flow and add proper tracking:

```html
<!-- Before: -->
<mat-card *ngFor="let item of menuItems" class="menu-item">

<!-- After: -->
@for (item of menuItems; track item.id) {
  <mat-card class="menu-item">
    ...
  </mat-card>
}
```

Explain: what problem does `track` solve, and what happens if you use `track $index` vs `track item.id`?
