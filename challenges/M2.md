# M2 — Angular Forms: Custom Validators & Multi-Step Form

**Level:** Mid
**Skills tested:** Reactive forms, custom `ValidatorFn`, cross-field validators, `AbstractControl`, `MatStepper`
**Files to edit:** `src/restaurant/reservation/reservation.component.ts`, `src/restaurant/reservation/reservation.component.html`, `src/restaurant/reservation/reservation.component.css`

---

## Context

The reservation form works but has two logic gaps:
1. A user can select a date in the past — the `MatDatepicker` doesn't prevent it and there is no validator.
2. The "special event" requirement (needed when guests > 6) is not enforced.

Additionally, the form is a single long page. Break it into a two-step wizard using `MatStepper`.

### Task 3 — Two-step form with MatStepper
Restructure the form into two steps:

**Step 1 — Contact Info** (its own `FormGroup`):
- `name` (with existing validators)
- `email` (with existing validators)

**Step 2 — Booking Details** (its own `FormGroup`, with `specialEventValidator` applied):
- `date` (with `futureDateValidator`)
- `time`
- `guests`
- `specialEvent` checkbox

Requirements:
- Use `MatStepperModule` with `linear="true"` — step 2 is locked until step 1 is valid
- If the user goes back to step 1 and changes their email, step 2 values must be preserved
- The submit button only appears on step 2
- Add "Back" and "Next" buttons appropriately
