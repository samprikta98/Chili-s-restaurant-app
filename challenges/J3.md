## Tasks

### Task 1 — Cart data structure
Add a `cartItems` property to `MenuComponent`. It should store the items the user has added, allowing duplicates (adding the same item twice means two entries).

Define a `CartItem` interface that extends `MenuItem` with a `quantity: number` field. Instead of duplicating items, use quantity to track how many of each item are in the cart.

### Task 2 — Add to cart
Implement `addToCart(item: MenuItem)`:
- If the item is already in the cart, increment its quantity
- If not, add it with `quantity: 1`
- Do not mutate existing cart items directly — create a new array or new object (immutable update pattern)

### Task 3 — Per-item badge
On each menu card, add an "Add to Cart" button. Next to or on the button, show a badge displaying how many of that item are in the cart (show `0` or hide the badge entirely when count is 0).

### Task 4 — Running total
Below the menu list, display:
```
Cart total: $XX.XX  (N items)
```
Compute the total using `Array.prototype.reduce`. Do not store the total as a separate property — derive it in the template or via a getter.

### Task 5 — Clear cart
Add a "Clear Cart" button that resets `cartItems` to an empty array. Disable it when the cart is empty.

## Acceptance criteria

- [ ] Clicking "Add to Cart" multiple times increments the quantity, not duplicates the item

- [ ] No `console.log` calls remain
